# Computer Architecture + Operating System

## 학습 목적

『혼자 공부하는 컴퓨터구조 + 운영체제』를 1회독하며 정리한 내용을 실제 Linux/C 실습으로 체감하는 것이 목표입니다.

단순히 책 내용을 요약하는 데서 끝내지 않고, 프로그램 실행, 메모리 구조, 프로세스, 스레드, 파일 시스템, 시스템 콜을 작은 실습으로 검증할 계획입니다.

## 핵심 개념

- CPU와 명령어
- 레지스터
- 메모리 계층 구조
- 캐시
- 보조기억장치
- 프로세스
- 스레드
- CPU 스케줄링
- 동기화
- 교착 상태
- 가상 메모리
- 파일 시스템
- 시스템 콜

- [02_operating_system.md](02_operating_system.md): 프로세스, 스레드, 스케줄링, 동기화, 메모리, 파일 시스템

## 실습 계획

- 프로그램 실행 과정 확인
- 스택/힙/데이터 영역 등 메모리 구조 확인
- 프로세스와 스레드 차이 실습
- Race Condition과 Mutex 확인
- 가상 메모리와 페이지 폴트 체감
- 파일 디스크립터와 시스템 콜 추적
- CPU 스케줄링 알고리즘 시뮬레이션
