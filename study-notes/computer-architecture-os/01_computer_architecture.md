# 01. Computer Architecture

## 학습 목적

컴퓨터가 명령어를 해석하고 실행하는 흐름을 이해하고, 이후 C/Linux 실습에서 프로그램 실행과 메모리 구조를 확인하기 위한 기초를 정리합니다.

## 핵심 개념

- CPU와 명령어
- 레지스터
- 메모리 계층 구조
- 캐시
- 보조기억장치

## 정리 내용

CPU는 명령어를 인출하고 해석하고 실행합니다. 레지스터는 CPU 내부에서 빠르게 접근하는 저장 공간이며, 메모리 계층 구조는 속도와 용량의 균형을 이해하는 데 중요합니다.

캐시는 자주 사용하는 데이터를 가까운 곳에 두어 접근 시간을 줄이는 구조입니다. 보조기억장치는 프로그램과 데이터를 장기적으로 저장하며, 운영체제의 파일 시스템과 연결됩니다.

## 실습 계획

- C 프로그램의 실행 파일 생성 과정 확인
- 배열 접근 패턴에 따른 캐시 친화성 실험
- 메모리 주소 출력으로 스택/힙 영역 차이 관찰
