# 02. Operating System

## 학습 목적

운영체제가 프로세스, 스레드, 메모리, 파일 시스템을 어떻게 관리하는지 이해하고 C/Linux 실습으로 연결합니다.

## 핵심 개념

- 프로세스와 스레드
- CPU 스케줄링
- 동기화
- 교착 상태
- 가상 메모리
- 파일 시스템
- 시스템 콜

## 정리 내용

프로세스는 실행 중인 프로그램의 단위이고, 스레드는 프로세스 내부에서 실행 흐름을 나누는 단위입니다. 여러 실행 흐름이 공유 자원에 접근하면 Race Condition이 발생할 수 있으며, Mutex 같은 동기화 도구로 제어합니다.

가상 메모리는 프로그램이 연속된 큰 메모리를 사용하는 것처럼 보이게 해 주며, 실제 물리 메모리와 디스크 사이의 관리는 운영체제가 담당합니다. 파일 시스템과 시스템 콜은 사용자 프로그램이 운영체제 기능을 사용하는 통로입니다.

## 실습 계획

- `fork`, `exec`, `wait` 흐름 확인
- pthread 기반 스레드 생성 실습
- Mutex 유무에 따른 결과 비교
- 파일 디스크립터와 `read`, `write` 시스템 콜 추적
