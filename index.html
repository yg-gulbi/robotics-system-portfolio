Direction

프로젝트는 결과물로, 학습은 근거로

공부한 주제를 억지로 프로젝트라고 부르지 않습니다. 실제 하드웨어, 코드, 데모, 문서가 있는 결과물은 프로젝트로 두고, 아직 배우는 중인 내용은 어디까지 이해했고 무엇을 다음에 검증할지 분명하게 남깁니다.

Main Project

Self Balancing Robot

실제 하드웨어 / 코드 / 데모 근거 보유

Arduino-ROS 셀프 밸런싱 로봇

Arduino가 실시간 균형 제어와 안전 판단을 맡고, ROS/Gazebo는 시뮬레이션, SLAM, navigation workflow 검증에 사용한 중심 프로젝트입니다.

  • 실제 로봇이 약 1시간 standing balance를 유지
  • 10m 복도 주행과 간단한 실내 장애물 코스 주행 수행
  • RC PWM filtering, tilt cutoff, current clamp로 안전 계층 구성
  • ROS/Gazebo simulation, SLAM map generation, navigation command path 문서화
원본 프로젝트 저장소 포트폴리오 요약

Control Flow

센서에서 안전한 모터 명령까지

핵심은 상위 명령이 모터로 바로 가지 않게 하는 것입니다. RC 또는 ROS 쪽 의도는 Arduino의 balance loop와 safety layer를 거친 뒤 ODrive current command로 바뀝니다.

Other Facts

나를 설명하는 다른 사실들

저는 기계를 움직이게 만드는 일뿐 아니라, 생각을 문장으로 정리하는 일에도 오래 머무는 편입니다. 로봇을 만들 때도 회로, 코드, 제어식만 보지 않고 왜 그렇게 실패했는지, 무엇을 다음에 바꿔야 하는지 글로 남기려 합니다.

언어를 배우고, 클라이밍을 하며 몸의 감각을 살피고, 배운 것을 다시 정리하는 과정은 기술 밖의 시간이지만 결국 제가 문제를 보는 방식에 영향을 줍니다. 이 포트폴리오는 로보틱스 개발자로서의 기록이면서, 기록하고 배우는 사람으로서의 제 모습도 함께 남기는 공간입니다.
