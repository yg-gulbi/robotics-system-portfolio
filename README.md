# Robotics System Portfolio

로봇 SW와 임베디드 시스템 개발자를 목표로, 제어, 센서, ROS, 운영체제 기초를 공부하고 프로젝트로 연결하는 포트폴리오 저장소입니다.

이 저장소는 완성된 결과물만 나열하기보다, 어떤 기술을 공부했고 어떤 실습으로 검증하고 있으며 앞으로 어떤 방향으로 확장할지 정리하는 데 목적이 있습니다.

## Website

- GitHub Pages: https://yg-gulbi.github.io/robotics-system-portfolio/

## 저장소 이름 선택 이유

저장소 이름은 `robotics-system-portfolio`로 정했습니다. `robotics-sw-portfolio`보다 운영체제와 시스템 프로그래밍 기초까지 담기 좋고, `embedded-robotics-portfolio`보다 로봇 SW와 시스템 학습의 균형을 자연스럽게 보여줄 수 있기 때문입니다.

## 관심 분야

- 로봇 소프트웨어
- 임베디드 시스템
- 제어와 센서 데이터 처리
- ROS/Gazebo 기반 로봇 시뮬레이션
- 컴퓨터구조와 운영체제 기초
- C/Linux 기반 시스템 프로그래밍

## 핵심 기술 스택 요약

| 영역 | 현재 정리 방향 |
| --- | --- |
| Language | C, C++, Python, MATLAB을 프로젝트와 실습 단위로 정리 |
| Embedded | Arduino, IMU, 모터 제어, 통신 기초를 밸런스 로봇과 연결 |
| Robotics | ROS, ROS2, Gazebo, 센서 토픽, Navigation, SLAM을 학습 중 |
| Control | PID, 상태공간, LQR을 균형 로봇 제어 실험과 연결 |
| CS/System | 컴퓨터구조, 운영체제, 시스템 콜, 프로세스/스레드 개념을 C/Linux 실습으로 검증 예정 |

자세한 기술 수준은 [tech-stack/skill_matrix.md](tech-stack/skill_matrix.md)에 정리했습니다.

## 현재 학습 중인 내용

- 운영체제 개념을 C/Linux 실습으로 체감하기
- PID 제어 실험 결과를 밸런스 로봇 문서로 정리하기
- ROS/Gazebo 시뮬레이션과 실제 로봇 제어 흐름 연결하기
- GitHub README와 Notion 기록을 취업 포트폴리오 문체로 정리하기

## 앞으로의 방향

1. 밸런스 로봇 프로젝트를 센서, 제어, 모터, ROS 연동 단위로 나누어 문서화합니다.
2. 운영체제 학습 내용을 C/Linux 실습 결과와 연결합니다.
3. ROS/Gazebo 실습을 실제 로봇 프로젝트의 확장 근거로 정리합니다.
4. 언어 학습 기록은 기술 문서 읽기와 설명 능력으로 연결합니다.
